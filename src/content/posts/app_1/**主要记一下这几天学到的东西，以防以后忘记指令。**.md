## **主要记一下这几天学到的东西，以防以后忘记指令。**



⚠️**默认已经配置好了证书（系统级）！！！**

⚠️ **需要root机或者模拟器**

➡️电脑端打开reqable
➡️开启全局代理，并开始监听
➡️手机端reqable配置二级代理，将二级代理ip配置为电脑端监听ip
➡️然后手机端配置代理，127.0.0.1，端口为调试端口。
➡️然后打开APP测试能不能连通，如果电脑端有手机端APP转发的包，再下一步。
➡️然后手机端插线，电脑端adb连接。

```bash
adb shell
```

```bash
su
```

🌟**在此之前先清除所有的无关的iptables规则:**

```bash
iptables -t nat -F && iptables -t nat -X
```

```bash
iptables -t filter -F && iptables -t filter -X
```




```bash
iptables -t mangle -F && iptables -t mangle -X
```

🌟**之后，配置透明代理，抓取不走WiFi代理的流量包**
🌟**先查看特定APP的包名**

```bash
pm list packages | grep -i autonavi
```


🌟**输出的包名用于下一个指令**

```bash
dumpsys package 包名 | grep userId=
```


🌟**获取id，在用于下一个指令**


```bash
iptables -t nat -A OUTPUT -p tcp -m owner --uid-owner <APP id> -j DNAT --to <手机端代理的ip和端口>
```

🔚到此透明代理配置完毕。然后再查看电脑端是否出现一些没有的包。这里举个例子**高德地图app**。很容易发现配置完后会出现很多没有的包



⏰如果出现ssl pinning 需要装个xposed或者LSPosed，然后装载JustTrustMe模块就行。GitHub都有。当然，还可以frida 或者xposed hook app的网络函数，这个就一劳永逸了，但是难度较高，这里先不记载。
